# Security Policy

## Supported Version

- 현재는 `main` 브랜치 최신 상태만 지원합니다.

## Reporting a Vulnerability

- 보안 취약점이 의심되면 공개 GitHub Issue에 올리지 마세요.
- 가능하면 GitHub의 private vulnerability report 또는 security advisory draft 기능으로 비공개 제보를 남겨 주세요.
- 그 기능을 사용할 수 없으면 저장소 소유자 GitHub 계정(`siyaness`)으로 비공개 연락을 먼저 시도해 주세요.

제보에 포함되면 좋은 내용:

- 영향받는 경로 또는 기능
- 재현 단계
- 예상 영향 범위
- 가능한 임시 완화 방법

확인 후에는 재현 여부를 검토하고, 필요하면 우선 완화 조치부터 진행합니다.
