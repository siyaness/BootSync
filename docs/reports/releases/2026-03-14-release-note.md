# BootSync Release Note

기준일: 2026-03-14

## 1. 문서 목적

이 문서는 최근 BootSync 프론트 전환과 UX 안정화 작업을 한 번에 공유하기 위한 요약 문서다.

- 무엇이 바뀌었는지
- 왜 바꿨는지
- 어떤 흐름이 현재 정상 동작하는지
- 남은 리스크가 무엇인지

를 빠르게 확인할 수 있도록 정리했다.

## 2. 이번 변경의 큰 방향

- 옛 프론트 의존을 걷어내고 React `/app` 프론트를 사용자 기준 단일 화면으로 정리
- 출결 관리와 학습 노트 화면의 실제 사용성 개선
- recovery email 인증/재발송 흐름 정리
- 문서, 명세, 테스트, 코드 상태를 현재 구현에 맞게 다시 동기화

## 3. 주요 변경 사항

### 3-1. 프론트 구조 정리

- 사용자 화면 기준으로 옛 SSR 프론트는 제거하고 React `/app` 구조를 주력 경로로 정리했다.
- 옛 URL은 외부 링크 호환을 위해 최신 `/app/...` 경로로 리다이렉트만 남겼다.
- 문서 허브는 `docs` 폴더 기준으로 재정리했다.

### 3-2. 출결 관리 UX 개선

- 달력 월 이동 시 레이아웃이 흔들리던 문제를 줄이기 위해 42칸 고정 캘린더 구조로 정리했다.
- 오른쪽 빠른 수정 패널의 레이아웃 흔들림, 버튼 위치 밀림, 불필요한 안내 배너를 정리했다.
- `기록 삭제`를 더 빠르게 접근할 수 있도록 패널 상단에 배치했다.
- 메모를 입력한 뒤 상태를 바꿨을 때 저장 흐름이 끊기지 않도록 정리했다.
- 주말 날짜도 명세 기준에 맞춰 사용자가 직접 저장할 수 있게 유지했다.

### 3-3. 학습 노트 흐름 개선

- 새 학습 노트 저장 후 상세 화면이 아니라 목록으로 돌아가도록 바꿨다.
- 태그 입력 누락 버그를 수정했다.
- 검색/태그 필터를 URL 쿼리 단일 소스로 정리했다.
- 필터가 걸린 상태에서도 사이드바 `학습 노트` 초기화와 다른 메뉴 이동이 멈추지 않도록 정리했다.

### 3-4. 복구 이메일 흐름 개선

- `pending` 인증 대상이 있으면 설정 화면에서 `인증 메일 재발송`이 보이도록 정리했다.
- 복구 이메일 변경 직후 성공 카드에서도 `로컬 확인 링크`와 `인증 메일 재발송`을 함께 제공한다.
- 재발송 cooldown은 `1분`으로 조정했다.
- 프론트에서도 재발송 직후 즉시 cooldown 안내와 버튼 비활성화가 보이도록 맞췄다.

### 3-5. 문서/정합성 정리

- README, 명세, 추적 문서, 작업 기록을 현재 코드 상태에 맞게 다시 정리했다.
- 최종 점검 상태는 `reports/checkpoints/final-checkpoint.md`로 따로 요약했다.
- 프론트 소스 디렉터리 이름도 `Front-End`에서 `frontend`로 정리해 실무형 경로 규칙에 맞췄다.

## 4. 현재 정상 동작으로 확인된 흐름

- 로그인 / 회원가입 / 대시보드 진입
- 출결 등록 / 수정 / 삭제 / 메모 저장
- 주말 출결 입력
- 학습 노트 작성 / 수정 / 삭제 / 태그 검색
- 복구 이메일 등록 / pending 상태 표시 / 로컬 확인 링크 / 재발송 / cooldown
- 옛 URL에서 최신 `/app` 경로로의 호환 리다이렉트

## 5. 검증 이력

자동 검증:

- `npm run build`
- `.\gradlew.bat compileJava compileTestJava`
- `.\gradlew.bat test --rerun-tasks`
- `.\gradlew.bat processResources --rerun-tasks`

수동 검증:

- 사용자 시나리오 기준 로그인, 출결, 학습 노트, 설정, recovery email 재발송 테스트 완료

## 6. 현재 남은 리스크

- 운영 SMTP 실메일 smoke test는 실제 운영 증적이 아직 필요하다.
- 로컬 rehearsal은 정리됐지만, purge 스케줄 첫 실행과 운영 AWS 자격증명 기준 S3 업로드, prod-like 복원 증적은 아직 남아 있다.
- 릴리즈 직전에는 데스크톱 기준 최종 수동 QA를 한 번 더 권장한다.
